# A2A Protocol v0.3.0 — Reference Guide

> Agent2Agent (A2A) is an open standard for interoperability between AI agents built with different frameworks, languages, or by different vendors.
> Official spec: https://a2a-protocol.org/v0.3.0/specification/

## Core Concepts

| Concept | Description |
|---|---|
| **A2A Client** | Initiates requests to an A2A Server (e.g., agent3 is a client of agent1/agent2) |
| **A2A Server** | Exposes an A2A-compliant HTTP endpoint to process tasks (e.g., agent1, agent2, agent3 are all servers) |
| **Agent Card** | JSON metadata at `/.well-known/agent-card.json` describing identity, capabilities, skills, and endpoint |
| **Task** | Stateful unit of work with lifecycle: `submitted → working → completed/failed/canceled/rejected` |
| **Message** | A communication turn (role: "user" or "agent") containing one or more Parts |
| **Part** | Smallest content unit: `TextPart`, `FilePart`, `DataPart` |
| **Artifact** | Output generated by agent (e.g., a file, JSON) |
| **Context** | Server-generated ID grouping related tasks |
| **Streaming (SSE)** | Real-time updates via Server-Sent Events (`message/stream`) |
| **Push Notifications** | Async updates via webhook POST to client URL |

## Transport

- **Protocol**: HTTP(S) only — all A2A calls are HTTP POST requests
- **Format**: JSON-RPC 2.0
- **Content-Type**: `application/json` (request/response) or `text/event-stream` (SSE)

## Agent Card Structure

```json
{
  "id": "unique-agent-id",
  "name": "Human Name",
  "description": "What this agent does",
  "version": "1.0.0",
  "url": "http://host:port",
  "capabilities": {
    "streaming": false,           // SSE support
    "pushNotifications": false    // Webhook support
  },
  "defaultInputModes": ["text"],
  "defaultOutputModes": ["text"],
  "authentication": null,         // null = no auth (demo only)
  "skills": [
    {
      "id": "skill_id",
      "name": "Skill Name",
      "description": "Skill description (used for routing decisions)",
      "tags": ["tag"],
      "examples": ["Example prompt"]
    }
  ]
}
```

**Discovery**: `GET https://{host}/.well-known/agent-card.json` (no auth by default)

## RPC Methods

| Method | HTTP | Purpose | Implemented Here |
|---|---|---|---|
| `message/send` | POST | Send message, get Task or Message back | ✅ All agents |
| `message/stream` | POST | Send message, receive SSE stream | ❌ (streaming: false) |
| `tasks/get` | POST | Poll task state by ID | ✅ (via SDK) |
| `tasks/cancel` | POST | Cancel in-progress task | ✅ (via SDK) |
| `tasks/pushNotificationConfig/set` | POST | Set webhook for task updates | ❌ |
| `tasks/pushNotificationConfig/get` | POST | Retrieve webhook config | ❌ |
| `tasks/resubscribe` | POST | Re-attach to SSE stream | ❌ |
| `agent/getAuthenticatedExtendedCard` | POST | Get authenticated agent card | ❌ |

## Task Lifecycle States

| State | Terminal? | Meaning |
|---|---|---|
| `submitted` | No | Task received, not yet processed |
| `working` | No | Agent is processing |
| `input-required` | No | Agent needs more info from user |
| `completed` | Yes | Successfully finished |
| `failed` | Yes | Unrecoverable error |
| `canceled` | Yes | Cancelled by client |
| `rejected` | Yes | Server refused to process |

> **Rule**: Once a task reaches a terminal state, it CANNOT be restarted. Send a new `message/send` to start fresh.

## Error Codes

| Code | Name | When |
|---|---|---|
| -32700 | Parse error | Invalid JSON |
| -32600 | Invalid request | Not valid JSON-RPC |
| -32601 | Method not found | Unknown method |
| -32602 | Invalid params | Wrong params |
| -32603 | Internal error | Server exception |
| -32001 | TaskNotFoundError | Task ID doesn't exist |
| -32002 | TaskNotCancelableError | Task already terminal |
| -32003 | PushNotificationNotSupportedError | Capabilities.pushNotifications=false |
| -32004 | UnsupportedOperationError | Feature not supported |
| -32005 | ContentTypeNotSupportedError | Unsupported media type in parts |
| -32006 | InvalidAgentResponseError | Agent returned invalid data |
| -32007 | AuthenticatedExtendedCardNotConfiguredError | Extended card not available |

## Authentication (A2A §4)

A2A defers auth to the HTTP transport layer. Auth info is NOT in JSON-RPC payloads.
- **In this project**: No auth (demo only, Docker-internal network)
- **Production**: Declare auth schemes in `AgentCard.authentication`, use Bearer tokens or mTLS

## Relationship to MCP

- **MCP (Model Context Protocol)**: Agent ↔ Tools/Resources (vertical — connects LLM to capabilities)
- **A2A**: Agent ↔ Agent (horizontal — connects agents to each other)
- They are complementary — an A2A agent can internally use MCP tools

## Common Patterns in This Codebase

### Checking agent capabilities before calling

```python
card = await fetch_agent_card(url)
if card.capabilities.streaming:
    # use message/stream
else:
    # use message/send + poll
```

### A2A ClientFactory usage
```python
client = await ClientFactory.connect(agent_url, client_config=ClientConfig(httpx_client=...))
async for event in client.send_message(message):
    ...
await client.close()
```

### Agent Card compliance checklist

- [ ] `id` is unique across all agents
- [ ] `url` matches Docker service DNS name + port
- [ ] `skills` descriptions are accurate (used for LLM routing)
- [ ] `capabilities.streaming` is set to correct value
- [ ] Agent Card is served at `/.well-known/agent-card.json`
