# Infrastructure

## Container Topology

```
Host Machine
├── Docker Engine
│   └── Compose Network: a2a-network (bridge)
│       ├── agent1   (build: ./agent1)  → host :8001 → container :8000
│       ├── agent2   (build: ./agent2)  → host :8002 → container :8000
│       ├── agent3   (build: ./agent3)  → host :8003 → container :8000
│       └── inspector (a2aprotocol/inspector:latest) → host :4000 → container :8080
```

## docker-compose.yml Summary

```yaml
version: '3.8'
services:
  agent1:
    build: ./agent1
    ports: ["8001:8000"]
    environment: [GEMINI_API_KEY]
    volumes: [./token.json:/app/token.json]
    networks: [a2a-network]         # NOTE: "netwo" typo in source — agent1 may not join network

  agent2:
    build: ./agent2
    ports: ["8002:8000"]
    environment: [GEMINI_API_KEY]
    volumes: [./token.json:/app/token.json]
    networks: [a2a-network]

  agent3:
    build: ./agent3
    ports: ["8003:8000"]
    environment: [GEMINI_API_KEY]
    networks: [a2a-network]

  inspector:
    image: a2aprotocol/inspector:latest
    ports: ["4000:8080"]
    networks: [a2a-network]
```

> **Known issue**: The `agent1` service has a typo `netwo:` (missing `rks`). This means agent1 may not be attached to `a2a-network`, preventing agent3 from reaching it by service name. Fix: change `netwo:` → `networks:` in `docker-compose.yml`.

## Dockerfile Pattern

All three agents use an identical Dockerfile:

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

The `langgraph_agent_template` Dockerfile adds a `PYTHONDONTWRITEBYTECODE=1` env var.

## Environment Configuration

| Variable | Services | Purpose | Required |
|---|---|---|---|
| `GEMINI_API_KEY` | agent1, agent2, agent3 | Authenticates to Google Gemini API | Yes |

**How to set up**:
```bash
cp .env.example .env     # (no .env.example exists yet — create manually)
echo "GEMINI_API_KEY=your-key-here" > .env
```

## Google OAuth2 Setup

Agents 1 and 2 require a pre-authenticated `token.json` OAuth2 refresh token:
```
project-root/
├── credentials.json   # OAuth Client ID (desktop) from Google Cloud Console
├── token.json         # Generated by: python generate_token.py
```

The `token.json` is bind-mounted into agent1 and agent2 containers at `/app/token.json`.
Agent3 does NOT need `token.json` (it only calls Gemini, not Google APIs).

## Local Development Setup

```bash
# 1. Clone and navigate
cd a2a_demo

# 2. Set environment
echo "GEMINI_API_KEY=your-key" > .env

# 3. Generate Google token (first time only)
python generate_token.py   # Opens browser, saves token.json

# 4. Build and run all services
docker-compose up -d --build

# 5. Verify containers
docker-compose ps

# 6. Open inspector
# http://localhost:4000
```
