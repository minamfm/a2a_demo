# Deployment Guide

## Build and Run

```bash
# Build all images and start in background
docker-compose up -d --build

# Wait ~5-10 seconds for agents to initialize, then check
docker-compose ps
```

Expected output:
```
NAME        SERVICE     STATUS    PORTS
agent1      agent1      running   0.0.0.0:8001->8000/tcp
agent2      agent2      running   0.0.0.0:8002->8000/tcp
agent3      agent3      running   0.0.0.0:8003->8000/tcp
inspector   inspector   running   0.0.0.0:4000->8080/tcp
```

## Pre-Deployment Checklist

- [ ] `.env` file exists with `GEMINI_API_KEY=...`
- [ ] `token.json` exists at project root (generated by `python generate_token.py`)
- [ ] `credentials.json` exists at project root (from Google Cloud Console)
- [ ] Docker Engine is running
- [ ] Ports 8001, 8002, 8003, 4000 are free on host

## Known Bug: agent1 Networking

The `docker-compose.yml` has a typo in agent1's config:
```yaml
# BUG: "netwo:" instead of "networks:"
agent1:
  netwo:        # <-- should be "networks:"
    - a2a-network
```

**Fix**: Edit `docker-compose.yml` line 12, change `netwo:` → `networks:`.
Until fixed, agent3 cannot discover agent1 by its Docker service name.

## Stopping and Cleanup

```bash
# Stop all containers
docker-compose down

# Stop and remove volumes
docker-compose down -v

# Rebuild a single agent
docker-compose up -d --build agent1
```

## Verification Steps

### Step 1: Verify Agent Cards
```bash
curl http://localhost:8001/.well-known/agent-card.json
curl http://localhost:8002/.well-known/agent-card.json
curl http://localhost:8003/.well-known/agent-card.json
```
Expected: JSON with agent name, skills, URL.

### Step 2: Test via A2A Inspector
1. Open `http://localhost:4000`
2. Set agent URL to `http://localhost:8003`
3. Send: `"Send an email to test@example.com saying hello"`
4. Expected: routing to agent1, confirmation email sent

### Step 3: Test Sheets
1. In Inspector, send: `"Create a spreadsheet named Test Sheet"`
2. Expected: agent3 routes to agent2, spreadsheet created with URL

### Step 4: Test Direct Response
1. Send: `"What is 2 + 2?"`
2. Expected: agent3 responds directly (no routing), returns `4`

## Rollback Procedure

Since there are no persistent data stores, rollback is:
```bash
docker-compose down
git checkout <previous-tag>
docker-compose up -d --build
```

## Adding a New Agent

1. Copy `langgraph_agent_template/` → `agent4/`
2. Edit `agent4/AgentCard.py` — define identity, URL (`http://agent4:8000`), skills
3. Edit `agent4/AgentExecutor.py` — implement tools and executor
4. Add to `docker-compose.yml`:
   ```yaml
   agent4:
     build: ./agent4
     ports: ["8004:8000"]
     environment: [GEMINI_API_KEY]
     networks: [a2a-network]
   ```
5. Run `docker-compose up -d --build agent4`
6. agent3 will discover agent4 automatically on next request

## Troubleshooting

| Problem | Command | Expected Fix |
|---|---|---|
| Container not starting | `docker-compose logs agent1` | Check for missing dependencies |
| `token.json` error | Check file exists at project root | Rerun `python generate_token.py` |
| agent3 returns no agents found | Fix `netwo:` → `networks:` typo | Rebuild + restart |
| Gemini API 401 | `echo $GEMINI_API_KEY` in container | Verify `.env` key is correct |
| Port conflict | `netstat -an | findstr 8001` | Stop conflicting process |
